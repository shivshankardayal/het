Prefix and Postfix Notataions

Prefix notation, also known as normal Polish notation (NPN), Łukasiewicz notation, Warsaw notation, Polish prefix notation or simply Polish notation(PN) was invented by Polish logician Jan Łukasiewicz. You can read about him at https://en.wikipedia.org/wiki/Jan_%C5%81ukasiewicz

Due to nature of prefix notation it does not need parenthesis as infix does(it will become clear soon). It is used heavily in LISP family of languages(this will take time). Scheme language which you have just used also belongs to same family of languages. It is not of much importance for us now except the fact that it will be used on GNU Guile prompt for calculations and verifying our calculations. The only point worth noting is that prefix notation can be easily and readily reduced to abstract syntax trees. I can give a link as to what it is but for now hold your curiosity a bit.

Anology can be drawn and prefix notation led to postfix notation. It is also known as Reverse Polish notation (RPN). As I have already told in this notation operator comes after both the operands. It was invented by Arthur Burks, Don Warren and Jesse Wright in 1954. Practcally it was used in HP-25 calculators and many other places. In programming it was used in stack based programming languages like Postscript.

Addition Exmaples

\(0 + 0 = 0\) \(0 + 1 = 1\) \(0 + 2 = 2\) \(0 + 3 = 3\) \(0 + 4 = 4\)
\(1 + 0 = 1\) \(1 + 1 = 2\) \(1 + 2 = 3\) \(1 + 3 = 4\) \(1 + 4 = 5\)
\(2 + 0 = 2\) \(2 + 1 = 3\) \(2 + 2 = 4\) \(2 + 3 = 5\) \(2 + 4 = 6\)

Observations

The most important observation is order of appearance of operands does not have effect on final value. This is known as commutative property of addition.

Note that it does not always hold true though it will a long while before I can show you.

The second observation is that adding zero to any number is not of any use as it does not change the number at all. Also, adding one increases value to next number and adding two increases the value to next-next number.

There are other properties of addition but all in due course of time. Patience is virtue.
