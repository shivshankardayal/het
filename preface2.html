Tools Used to Write This Book

This entire book has been typed using raw HTML5 technologies. I have used Emacs with web-mode. Material design has been used and I do not force a font on users except for headings. You can set default font in your browser and if you do not set then it will be rendered mostly in a serif font.

For rendering Mathematics, I have used MathJax. Since pages are small it does not really matter as very little math will be present on single page. I have tried to keep the UI sane and responsive in nature. But the problem is when I edit the pages they do cros A4 paper size. Keeping it in HTML is important as it can be scanned and indexed by a search-engine. That is why I will not prepare an index as that would be a herculean task.

For preparing images I have mostly used raw SVG, which are hand-coded. For some images I have used POV-Ray, which is a rayracer program.

Great folks of Mozilla have written great documentation on HTML5, CSS3, SVG along with Math authoring. For now bear the MathJax version. Eventually, I will run all this through phantomjs and render all math server side making the load blazing fast.

You would ask me what about index. This book will have no index. I rely on search engines to generate that.

There is one more reason to use raw HTML5, CSS3 and SVG is that frameworks come and go. Also, I wanted to learn these technologies in depth. So you see it is kind of selfish. I did use Sphinx documentation tool but recently with version 1.7.0 release several plugins broke. Then, I realized why Knuth froze his typesetting engine τεχ in the sense that new features are not allowed.
